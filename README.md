# 2026 연세화우 YB전 — 전자 아카이브

연세대학교 화우회 동문·재학생 예술 교류전의 전자 도록이자 행사 기록 아카이브입니다.
GitHub Pages에 그대로 올리면 바로 배포됩니다.

---

## 1. 배포 방법 (처음 한 번만)

1. GitHub에 새 저장소를 만듭니다. (예: `yonsei-whawoo/2026-yb-exhibition`)
2. 이 폴더(`site/`) 안의 내용물을 저장소 루트에 그대로 업로드합니다.
   - `index.html`이 저장소 최상위에 바로 보여야 합니다. (하위 폴더에 넣지 마세요)
3. GitHub 저장소 → **Settings → Pages** → Source를 `main` 브랜치, `/ (root)`로 설정합니다.
4. 몇 분 후 `https://[아이디].github.io/[저장소이름]/` 로 접속하면 사이트가 보입니다.
5. **`assets/js/config.js` 파일을 열어 `GITHUB_REPO` 값을 실제 저장소로 채워주세요.**
   ```js
   GITHUB_REPO: "yonsei-whawoo/2026-yb-exhibition",
   ```
   이 값을 채워야 "사진만 올리면 자동으로 이어서 표시되는" 기능이 켜집니다. (2번 항목 참고)
   비워두면(`""`) 아래 매니페스트 파일 목록으로만 표시됩니다.

---

## 2. 나중에 파티/현장 사진을 추가하고 싶을 때

**가장 쉬운 방법 — 코드 수정 없이 사진만 올리기**

- `assets/img/party/` 폴더에 사진 파일(jpg/png)을 추가로 올리기만 하면,
  사이트가 GitHub에서 그 폴더 목록을 직접 읽어와 자동으로 화면에 추가합니다.
- `assets/img/scenes/` 폴더도 동일하게 작동합니다. (전시 현장 사진 섹션)
- 파일명은 자유롭게 지어도 되고, 정렬은 파일명 알파벳/숫자 순으로 표시됩니다.
  예: `party-05.jpg`, `2026-07-06-party.jpg` 등 무엇이든 가능합니다.
- 사진 용량이 너무 크면(10MB 이상) 로딩이 느려지니, 업로드 전에 2000px 폭 정도로
  리사이즈하는 것을 권장합니다.

**주의할 점**

- 이 기능은 `assets/js/config.js`의 `GITHUB_REPO` 값이 채워져 있어야 작동합니다.
- 저장소가 **Public(공개)** 이어야 별도 설정 없이 작동합니다. Private 저장소라면
  GitHub API가 인증 없이는 폴더 목록을 읽지 못해 자동 표시가 되지 않습니다.
- GitHub API는 비로그인 요청을 시간당 60회로 제한합니다. 개인 아카이브 사이트
  방문량 수준에서는 문제되지 않지만, 혹시 사진이 갑자기 안 보이면 몇 분 후
  새로고침해보세요. (그 사이엔 아래 매니페스트로 자동 대체됩니다)
- 매니페스트 파일(`assets/data/party-manifest.json`, `scenes-manifest.json`)은
  저장소를 아직 설정하지 않았거나 GitHub API가 응답하지 않을 때를 대비한
  예비용 목록입니다. 평소에는 신경 쓰지 않아도 됩니다.

## 3. 작품 정보(캡션)를 고치거나 추가하고 싶을 때

모든 작품 데이터는 **`assets/data/works.json`** 한 파일에 모여 있습니다.
코드를 몰라도 이 파일 안에서 텍스트만 고치면 사이트에 바로 반영됩니다.

```json
{
  "id": "alumni-01",
  "cohortNum": 15,        ← 정렬 기준 숫자 (기수)
  "cohort": "15기",       ← 화면에 보이는 기수 표기
  "name": "이재우",
  "major": "건축공학과",
  "year": "82학번",
  "title": "STELLA",
  "material": "유화",
  "size": "45×38 cm",
  "image": "assets/img/works/alumni/alumni-01.jpg",   ← 원본(모달용) 이미지 경로
  "thumb": "assets/img/works/alumni/thumb/alumni-01.jpg" ← 목록용 썸네일 경로
}
```

- 동문 작품은 `alumni` 배열, 재학생 작품은 `student` 배열 안에 있습니다.
- 정렬은 `cohortNum` 숫자를 기준으로 자동 오름차순 정렬됩니다. 새 작가를
  추가할 때 배열 순서를 신경 쓰지 않아도 됩니다.
- 새 작품을 추가하려면: ① 이미지를 `assets/img/works/alumni(또는 student)/`에 넣고,
  ② 썸네일도 같은 이름으로 `thumb/` 하위 폴더에 넣은 뒤, ③ works.json에 위 형식대로
  한 항목을 추가하면 됩니다.

## 4. 이미지·데이터 매칭 방식

첨부해주신 자료를 다음 기준으로 대조했습니다.

1. **동문 작품 20개 이미지 파일명 ↔ 명단 엑셀(`2026 화우미전 명단.xlsx`) 이름** 매칭
2. **캡션 PDF(`26 동문미전 캡션.pdf`)** 로 제목·재료·크기 2차 검증 (엑셀과 PDF 내용이
   서로 일치함을 확인했습니다)
3. 박정서 작가는 파일명에 작품명이 함께 표기되어 있어(`박정서_귤과 황동 접시.jpg`,
   `박정서_피클단지와 호박.jpg`) 2개 작품으로 정확히 분리 처리했습니다.
4. 재학생 최유진 동명이인 2명도 파일명의 작품명(`최유진_일탈`, `최유진_피어나리라`)
   기준으로 정확히 분리 처리했습니다. (전공·학번도 서로 달라 이중으로 확인됨)
5. 최종 인원수도 교차 검증했습니다 — 동문 19명, 재학생 13명(동명이인 포함),
   출품작 39점 (명단 엑셀 하단 합계와 일치).

## 5. ⚠ 확인이 필요한 사항

아래 3건은 임의로 추정하지 않고 있는 그대로 반영했습니다. 확인 후
`works.json`에서 직접 고쳐주세요.

| 작가 | 상황 |
|---|---|
| **이준택 (27기)** | 명단·캡션 기준 3점(올려보다·비춰보다·내려보다) 출품이나, 제공된 이미지는 3점이 함께 걸린 전시 전경 사진 1장뿐입니다. 개별 작품 단독 사진이 있다면 추가로 보내주세요. |
| **진연정 (32기)** | 명단·캡션 기준 4점(이화동 그 골목 1~4) 출품이나, 제공된 이미지는 4점이 함께 걸린 전시 전경 사진 1장뿐입니다. 개별 작품 단독 사진이 있다면 추가로 보내주세요. |
| **윤현정 (51.5기)** | 명단·캡션 기준 2점 출품이며, 제공된 이미지 1장에 2점이 나란히 액자로 걸려 있어 그대로 사용했습니다. 개별로 분리하고 싶다면 사진을 나눠서 다시 보내주세요. |

현재는 각 항목에 "확인필요" 배지, 또는 "n점" 배지가 붙어 있고, 모달 상세보기에도
동일한 안내 문구가 함께 표시됩니다. 확인이 끝나면 `works.json`에서
`needsReview` / `reviewNote` 항목을 지우면 배지가 사라집니다.

**참고 — About Whawoo 섹션 문구 관련**
`화우회 동아리 설명.png` 이미지 속 창립 관련 문구 중 특정 공간명(구체적 고유명사)이
저해상도로 인해 OCR·육안 판독이 완전히 확실하지 않아, 해당 고유명사는 웹사이트
본문에 넣지 않고 확실한 사실(1968년 9월 창립, 미술대학이 없는 연세대학교의
순수 아마추어 동아리, 1979년 첫 동문전)만으로 문구를 새로 작성했습니다. 원문
그대로의 표현이 필요하시면 원본 이미지를 보고 `index.html`의 About 섹션 문구를
직접 다듬어주세요.

## 6. 폴더 구조

```
site/
├─ index.html
├─ README.md
└─ assets/
   ├─ css/style.css
   ├─ js/
   │  ├─ config.js        ← GITHUB_REPO 등 설정값
   │  └─ main.js           ← 데이터 로딩, 탭, 라이트박스, 자동 로딩 로직
   ├─ data/
   │  ├─ works.json              ← 출품작 전체 데이터 (동문/재학생)
   │  ├─ awards.json              ← 수상 데이터
   │  ├─ party-manifest.json      ← 파티 사진 예비 목록
   │  └─ scenes-manifest.json     ← 현장 사진 예비 목록
   └─ img/
      ├─ hero/            ← 히어로 대표 사진
      ├─ about/           ← 화우회 과거 사진 2장
      ├─ scenes/          ← 전시 현장 사진 (자동 확장 가능)
      ├─ works/
      │  ├─ alumni/ , alumni/thumb/
      │  └─ student/ , student/thumb/
      ├─ awards/          ← 수상 사진 3장
      ├─ party/           ← 파티 사진 (자동 확장 가능)
      └─ misc/            ← 로고, QR, 포스터, 카탈로그 이미지
```

원본 사진은 총 192MB였으나, 웹 게시에 적합하도록 리사이즈·압축하여
전체 약 24MB로 최적화했습니다 (화질 저하는 거의 느껴지지 않는 수준입니다).

## 7. 기타 참고

- 갤러리 위치는 요청하신 대로 한 줄짜리 텍스트 정보로만 표기했고, 별도의
  지도·길찾기 기능은 넣지 않았습니다.
- 상단 플로팅 섹션 네비게이션(점 메뉴)은 제거했습니다. 대신 왼쪽 상단의
  작은 브랜드 바(로고+제목)만 스크롤 내내 유지됩니다.
- 폰트는 Google Fonts(Song Myung, Fraunces, Noto Sans KR, IBM Plex Mono)를
  CDN에서 불러옵니다. 인터넷 연결이 없으면 시스템 기본 폰트로 자연스럽게
  대체됩니다.
